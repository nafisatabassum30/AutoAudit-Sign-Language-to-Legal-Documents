# BdSL FIR LLM — Bangla Legal Complaint Generator

Part of the **AutoAudit** pipeline. This module takes raw Bangla text output
from the Spatial-Temporal GNN (sign language recognition) and converts it into
structured, legally compliant **FIR (First Information Report)** documents.

---

## Architecture

```
Informal Bangla Text (from GNN)
          │
          ▼
  ┌───────────────────┐
  │  Prompt Template  │   Alpaca-style instruction prompt
  │  (Bangla)         │
  └────────┬──────────┘
           │
           ▼
  ┌───────────────────┐
  │  Llama 3 8B       │   Base model (4-bit NF4 quantised)
  │  + QLoRA Adapter  │   LoRA rank=16, α=32
  └────────┬──────────┘
           │
           ▼
  ┌───────────────────┐
  │  FIR Text Output  │   Structured Bangla legal complaint
  └────────┬──────────┘
           │
           ▼
  ┌───────────────────┐
  │  Entity Extractor │   Complainant, Accused, Location,
  │  (regex + rules)  │   Date, Time, Legal Section
  └───────────────────┘
```

## Quick Start

### 1. Install dependencies

```bash
cd llm
pip install -r requirements.txt
```

For faster training (2–5× speedup), install Unsloth:
```bash
pip install "unsloth[cu121-ampere-torch220]"
# or
pip install "unsloth[colab-new] @ git+https://github.com/unslothai/unsloth.git"
```

### 2. Set environment variables

```bash
cp .env.example .env
# Edit .env — add your HuggingFace token for Llama 3 access
```

> **HuggingFace Token**: Llama 3 is a gated model. Request access at
> [meta-llama/Meta-Llama-3-8B](https://huggingface.co/meta-llama/Meta-Llama-3-8B)
> and set `HUGGING_FACE_HUB_TOKEN` in `.env`.

### 3. Generate synthetic training data

```bash
python scripts/generate_data.py --n_samples 3000
```

This creates `data/synthetic/train.json`, `validation.json`, `test.json`.

### 4. Fine-tune the model

```bash
python src/train.py --config configs/training_config.yaml
```

Training outputs are saved to `models/checkpoints/`.

For custom hyperparameters:
```bash
python src/train.py \
    --config configs/training_config.yaml \
    --learning_rate 1e-4 \
    --num_train_epochs 5
```

### 5. Run inference

```bash
python src/inference.py \
    --text "গতকাল রাত ১০টায় মিরপুর ১০ এলাকায় বাদল মিয়া আমার মোবাইল ছিনিয়ে নিয়েছে।"
```

With streaming:
```bash
python src/inference.py \
    --text "..." \
    --stream
```

### 6. Start the API server

```bash
python api/app.py
```

The server starts on port 5000 by default.

## API Reference

### `POST /generate`

Convert informal Bangla text to a structured FIR.

**Request**
```json
{
  "text": "গতকাল রাত ১০টায় মিরপুর ১০ এলাকায় বাদল মিয়া আমার মোবাইল ছিনিয়ে নিয়েছে।"
}
```

**Response**
```json
{
  "input": "...",
  "fir_text": "বাংলাদেশ পুলিশ\nপ্রথম তথ্য বিবরণী...",
  "entities": {
    "complainant": "...",
    "accused": "বাদল মিয়া",
    "location": "মিরপুর ১০",
    "date": "...",
    "time": "রাত ১০টা",
    "legal_section": "দণ্ডবিধি ধারা ৩৯২"
  },
  "latency_ms": 1234
}
```

### `POST /generate/batch`

```json
{ "texts": ["text1", "text2"] }
```

### `POST /generate/stream`

Server-Sent Events; streams tokens as they are generated.

### `GET /health`

Returns GPU info and model load status.

## Training Configuration

Key parameters in `configs/training_config.yaml`:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `model.model_name` | `unsloth/llama-3-8b-bnb-4bit` | Base model |
| `lora.r` | 16 | LoRA rank |
| `lora.lora_alpha` | 32 | LoRA alpha (effective LR scaling) |
| `num_train_epochs` | 3 | Training epochs |
| `per_device_train_batch_size` | 4 | Batch size per GPU |
| `gradient_accumulation_steps` | 4 | Effective batch = 16 |
| `learning_rate` | 2e-4 | Peak LR (cosine decay) |
| `max_seq_length` | 1024 | Max tokens per sample |

**GPU requirements**: 16 GB minimum (with 4-bit quantisation). 24 GB+ recommended.

## Evaluation

```bash
python scripts/evaluate.py \
    --model_path models/checkpoints/final_adapter \
    --data_path  data/synthetic/test.json
```

Metrics reported:
- **BLEU-4** — n-gram overlap with reference FIR
- **ROUGE-1/2/L** — recall-oriented overlap
- **Entity F1** — per-field accuracy (complainant, accused, location, etc.)
- **Field Coverage** — fraction of required FIR fields present

## Integration with AutoAudit Pipeline

The inference module exposes a single clean interface consumed by the upstream Flask gateway:

```python
from llm.src.inference import FIRGenerator, InferenceConfig

generator = FIRGenerator(InferenceConfig())
result = generator.generate_structured(bangla_text_from_gnn)
# result["fir_text"]    — full FIR document
# result["entities"]    — structured fields
```
