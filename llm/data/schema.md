# Bangla-FIR Dataset Schema

This document defines the data format expected by the LLM fine-tuning
pipeline in `llm/`. It is the contract between:

- **Upstream**: the ST-GNN sign-recognition stage, which produces raw,
  informal Bangla text from a sequence of BdSL gestures.
- **Downstream**: the fine-tuned LLM, which converts that raw text
  into a structured First Information Report (FIR).

## Raw training record (JSONL)

Each line of a raw dataset file (e.g. `data/seed_dataset.jsonl`) is a
JSON object with two keys:

```json
{
  "input_text": "আমার মানিব্যাগ চুরি হয়েছে উত্তরা বিকাল ৫টায়",
  "fir": {
    "date_of_occurrence": "০১/০৬/২০২৬",
    "time_of_occurrence": "বিকাল ৫টায়",
    "location": "উত্তরা, বাংলাদেশ",
    "offense_type": "চুরি",
    "complainant_name": "রহিম ইসলাম",
    "complainant_address": "উত্তরা, বাংলাদেশ",
    "complainant_contact": "01712345678",
    "accused": "অজ্ঞাত",
    "description": "অভিযোগকারীর মানিব্যাগ উত্তরা এলাকায় বিকাল ৫টার সময় চুরি হয়ে যায়।"
  }
}
```

- `input_text`: the raw Bangla sentence(s) as they would come out of
  the sign-recognition stage. Keep this informal/terse -- that's what
  the model will see in production.
- `fir`: the target structured document. All 9 fields defined in
  `src/fir_schema.py::FIR_FIELDS` **must** be present. Use
  `"উল্লেখ নেই"` for genuinely unknown fields, and `"অজ্ঞাত"`
  specifically for an unknown accused/suspect (this matches
  conventional Bangladeshi FIR phrasing).

## Where this data comes from

1. **Bootstrap / development**: `scripts/generate_seed_dataset.py`
   synthesizes examples like the one above from templates, so the
   training/eval pipeline can be built, unit-tested, and smoke-tested
   before real data exists. This is **not** a substitute for real
   data -- it exists purely to unblock engineering work.

2. **Real data (recommended path per the project plan)**:
   - Collect 1,000+ real (anonymized) FIR templates/examples from
     police records or BLAST (Bangladesh Legal Aid and Services
     Trust) or similar legal-aid organizations.
   - For each, write (or have a native Bangla speaker write) a short,
     informal sentence that mimics what a BdSL signer would convey
     for that incident (or, once the ST-GNN stage is trained, run
     real BdSL video through it to get authentic `input_text`
     samples and pair them with a matching FIR).
   - Store the results in the same JSONL format in `data/raw/` and
     point `--raw-data` at it (see `llm/README.md`).

3. **Data augmentation**: once you have a seed of real examples,
   `scripts/generate_seed_dataset.py` can be adapted (new
   `OFFENSES`/`LOCATIONS`/templates) to expand coverage of offense
   types, districts, and phrasing styles before committing to a full
   fine-tuning run.

## Processed / tokenized format

`src/data_preparation.py` turns raw JSONL into:

- `data/processed/train.jsonl`, `val.jsonl` -- the same
  `{input_text, fir}` records, just split (human-readable, used by
  `src/evaluate.py`).
- `data/processed/train_tokenized.jsonl`, `val_tokenized.jsonl` --
  `{input_ids, attention_mask, labels}` ready for the `Trainer`, where
  `labels` has prompt tokens masked to `-100` so the loss is only
  computed on the target FIR JSON.

This directory is regenerated by `src/train.py` on every run and is
git-ignored.
